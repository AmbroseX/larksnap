# 飞书新建文档接口调研（iflytek 私有化实测）

- 日期：2026-07-09
- 环境：`https://yf2ljykclb.xfchat.iflytek.com`（讯飞私有化）+ `feishu.cn`（公有云个人版），两边都实测
- 方法：feishu-doc-probe skill，Playwright 在已登录页面上下文录流量，人工点「新建 → 文档」抓包
- 结论一句话：**新建文档 = 一个 REST POST 调用，公私接口通用，只生成空文档；正文要另走现有粘贴通道**

---

## 1. 背景与目标

larksnap 现有编辑能力（append/insert/replace/delete）都要求传入**已存在**的飞书文档，
`runEditJob` 开头就校验，不认识就报 `edit_unsupported`；`feishu-api.ts` 里没有任何创建文档的接口。

本次目标：搞清这个私有化部署点「新建 → 文档」时打的是哪个内部接口、请求体和返回结构，
为后续给 larksnap 增加"新建文档"能力定位接口。

## 2. 创建接口

```
POST /space/api/explorer/v2/create/object/
Content-Type: application/x-www-form-urlencoded      ← 表单编码，不是 JSON
```

请求体（form）：

| 字段 | 实测值 | 说明 |
|---|---|---|
| `parent_token` | 空 | 空 = 我的空间根目录；填**文件夹节点 token** 则建到指定文件夹 |
| `type` | `22` | 22 = docx 新版文档（与 obj_type 对齐） |
| `name` | 空 | 空 = 未命名文档；可传标题 |
| `time_zone` | `Asia/Shanghai` | |
| `source` | `0` | |
| `ua_type` | `Web` | |
| `scene` | `space_create` | 从网盘首页新建的场景标识 |

原始 body：
`parent_token=&type=22&name=&time_zone=Asia%2FShanghai&source=0&ua_type=Web&scene=space_create`

## 3. 请求头 / CSRF

沿用 feishu-doc-probe 已知结论，POST 需要：

- `x-csrftoken`：取 Cookie **`_csrf_token`** 的值（**不是** `swp_csrf_token`）
- `doc-biz: Lark`
- `referer`：指向 `/drive/me/`（同源，feishu-proxy 天然满足）
- `request-id` / `x-request-id` / `x-tt-trace-id`：同一随机 id（可选，贴合页面形态更稳）

## 4. 返回结构

`200`，`code:0`，新文档 token 直接在返回里：

```jsonc
{
  "code": 0,
  "data": {
    "node_list": ["nodrzApGitKVMyCc1OORBq0NjNd"],
    "entities": { "nodes": { "nodrzApGitKVMyCc1OORBq0NjNd": {
      "obj_token": "doxrzYkKVV1F6FBJUVbY4GfSj6d",   // ← 文档本体 token：client_vars / edit 用这个
      "token":     "nodrzApGitKVMyCc1OORBq0NjNd",    // ← explorer 节点 token：列目录/移动/删除用
      "type": 22,
      "url": "https://.../docx/doxrzYkKVV1F6FBJUVbY4GfSj6d",
      "name": "",           // 未命名
      "create_time": 1783589550,
      "edit_time": 1783589550,
      "owner_id": "7459258065024451446"
    }}}
  }
}
```

- 新文档 URL = `/docx/{obj_token}`
- 两个 token 别搞混：`obj_token`（doxr… 开头）是文档本体，喂给 `client_vars`、edit；
  `token`（nodr… 开头）是 explorer 节点，用于目录/移动/删除。

## 5. 关键发现：创建 ≠ 写内容

抓包时创建后又敲了几个字，**敲字没有产生任何 REST 请求**——正文改动走协同 WebSocket
（`POST /space/api/pandora_ws/ws_ticket/` 拿票 + WS 长连推送），不走 HTTP。

所以 `create/object/` **只生成一篇空文档**。要"新建带内容的文档"，得两步：

1. 调 `create/object/` 建空文档，拿 `obj_token`
2. 拼出 `/docx/{obj_token}`，**复用现有 `edit append` 的 CDP 可信粘贴**把 Markdown 写进去

这正好接上现有路线 C（读靠接口、写靠跳转、验靠服务端），不需要逆向协同协议。

## 5.5 公有云 vs 私有化对照（feishu.cn 已实测）

feishu.cn 公有云（飞书个人版）用**同一个接口、同一套返回结构**，只有一处无关差异：

| 项 | iflytek 私有化 | feishu.cn 公有云 |
|---|---|---|
| 接口 | `POST /space/api/explorer/v2/create/object/` | 完全一样 |
| body 编码 | form-urlencoded | 一样 |
| 核心字段 | `parent_token/type=22/name/time_zone/source=0/ua_type=Web/scene=space_create` | 一样 |
| 额外字段 | 无 | 多 `ext_info={"platform":"web"}` |
| CSRF | `_csrf_token` → `x-csrftoken` | 一样 |
| 返回 | `obj_token`+`token`+`url=/docx/{obj_token}` | 结构一模一样 |
| 正文写入 | 协同 WS，不走 HTTP | 一样 |

- `ext_info` 是**可选**的：私有化没传也成功。larksnap 实现带不带都行，带上更贴合公有云页面形态。
- 结论：**创建接口公私通用，可放心封装一份走两边**（路径相对 origin，天然适配私有化域名）。
- 公有云返回样例：`obj_token=ZO3gdsJAWoWNJ8xpxi3cqcd7ncb` → `https://my.feishu.cn/docx/ZO3gdsJAWoWNJ8xpxi3cqcd7ncb`。

## 6. 对 larksnap 的落地建议（YAGNI，先不写代码）

- 创建接口可走现有 `feishu-proxy` 同源 POST 代发（和 `export/create` 一个路子），
  只需在 `src/background/feishu-api.ts` 新增一个 `createDocx({ parentToken?, name? })`，
  注意 body 是 form 编码而非 JSON。
- 新命令编排：`create/object/` → 得 `obj_token` → 交给现有 `runEditJob` 的 append 分支写内容 → 回读校验。
- CLI 侧可加 `edit new-doc --name --parent --content-md`（或独立 `new` 子命令），
  返回新文档 URL 给用户。

## 6.5 删除文档接口（顺带研究，用于清理测试文档）

抓包实测（iflytek 私有化），右键文档「删除 / 移入回收站」走：

```
POST /space/api/explorer/v3/remove/
Content-Type: application/json        ← 这个是 JSON，不是 form
body: {"tokens":["<节点token>"],"apply":1}
→ {"code":0,"data":{"task_id":"schedule|…"}}   // 异步任务，移入回收站
```

- `tokens` 用的是 **explorer 节点 token**（`nodr…` 开头，即 `createDocx` 返回的 `nodeToken`），
  **不是** `obj_token`。支持数组，可批量删。
- body 是 JSON，larksnap 要加删除能力可直接用现有 `feishuPost`（不用 form）。
- 是"移入回收站"，非彻底删除；回收站内彻底删/还原是另外的接口（未抓，暂不需要）。

## 7. 待验证项（本次未覆盖）

- **建到指定文件夹**：`parent_token` 填文件夹节点 token 是否即可（推测可，未实测）。
- **建其它类型**：`type` 换 wiki / sheet / bitable / mindnote 的取值（本次只测了 docx=22）。
- **在 wiki 知识库内新建**：wiki 场景 scene/接口可能不同（`space_create` 是网盘首页场景），需另抓。
- **建时直接带标题**：`name` 传值是否生效、是否需要额外接口设标题。
- ~~公有云差异~~：✅ 已测，feishu.cn 公有云接口/字段/返回与私有化一致（仅多可选 `ext_info`），见 5.5。
