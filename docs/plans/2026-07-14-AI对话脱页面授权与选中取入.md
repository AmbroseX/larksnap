# AI 对话：脱离整页授权 + 选中文字取入

日期：2026-07-14
状态：已编码完成（typecheck / 单测 / 构建通过），待真机验证
关联：007（AI 对话与流式总结）、006（侧边栏首页/对话双视图）

---

## 一、这次要解决的两个问题

1. **切到「AI 对话」也被要求授权域名**，用户困惑："任何页面都应该能对话，为什么还要授权？"
2. **希望鼠标选中网页文字后**，在 AI 对话里能自动把选中内容带进对话框，并在对话上方给出「总结 / 翻译 / …」这类一键推荐按钮。

---

## 二、根因分析（已核对代码）

### 问题 1 拆成两层

**第一层：横幅显示漏洞（已当场修掉）**

用户看到的「通过命令行（larksnap-fetch）后台抓取本网页，需先授权访问该域名」这条帯，是 `SidePanel.tsx` 里的 `webNeedsAuth` 横幅（`sidepanel.authBannerWeb`）。它本来只服务**首页（工具）视图**——给 larksnap-fetch 常驻进程在后台抓取当前页一个授权入口，跟 AI 对话毫无关系。

但这条横幅没有像 `<main>`、footer 那样跟随标签页显隐，所以切到对话页它还赖着不走，造成"连聊天都要授权"的误解。
已修：给两个授权横幅都加上 `view !== 'chat'` 的条件（`src/sidepanel/SidePanel.tsx`）。

**第二层：对话本身就是"先抓整页"（这次计划的重点）**

即使横幅隐藏了，真去点「立即总结」还是会弹授权。原因在设计：

- 对话的唯一入口是 `startSummary`（`ChatView.tsx:135`）。
- 它走 `SUMMARIZE_PREPARE` → `collectSource` → `webcopyPageMd(tabId, url)`（`summarize/index.ts:190`），**把整页正文抓下来**喂给模型。
- 读任意页面全文 = 需要该域名的**恒久 host 权限**（`manifest.json` 里 `optional_host_permissions: ['*://*/*']`），所以每次弹授权框。
- 后台 `chat-port.ts` 里，会话**必须**由 `start-summary` 携带 `sourceId` 创建，第一条消息就是整页正文（`kind:'source'`）；`ask` 只能追问已存在的会话。**没有"不抓页面、直接开聊"的通路**。
- 前端输入框 `disabled={!session || busy}`（`ChatView.tsx:412`），没有 session 就不能打字，而 session 只能靠总结产生。

**一句话：现在"聊天"和"抓整页"焊死在一起了。**

### 问题 2 的可行性（权限前提，已核对 manifest）

- `activeTab` 已常驻授予 → 用户点开/操作扩展的那一刻，读**当前标签**的内容不弹授权框。
- context menu 已有 `selection` 上下文（`context-menus.ts:75`）→ `info.selectionText` 天然可取，零权限。
- 没有 content script（`content_scripts: []`）。

结论：**取"选中的一小段文字"不需要恒久 host 权限**，可以做到零授权弹窗。真正需要授权的只有"抓整页"那一条路。

## 三、设计方案：把"会话"和"取页面"解耦

核心思路一句话：**默认是纯聊天（零授权），页面/选中内容只在你按下对应按钮时才作为"上下文"喂进去。**

### 3.1 纯聊天通路（零授权）

新增一条不碰 `collectSource` 的开聊路径：

- 后台 `chat-port.ts` 新增客户端消息 `start-chat`（或让 `ask` 在无 `sessionId` 时创建空会话）。会话第一条是**普通用户消息**，不是 `kind:'source'`，系统提示用通用聊天 `chatSystemPrompt(targetLang)`。
- 整个链路不调用 `webcopyPageMd`，因此**永不抓页面、永不弹授权**。
- 前端：
  - 空状态从"只有一个『立即总结』按钮"改成"可直接打字发送 + 下面挂几个可选动作"。
  - 输入框 `disabled` 去掉 `!session` 约束，改成只受 `busy` 约束——没有会话也能打第一句。
  - `send`/`sendText` 在无 session 时走 `start-chat` 建会话，之后照常 `ask`。

### 3.2 「总结当前页」变成一个显式动作（保留旧能力）

- 现有 `start-summary`（抓整页 → 总结）**保留**，但从"开聊的唯一门槛"降级为"一个明确的按钮：总结当前页"。
- 只有点它才会抓整页、才可能弹授权。用户想纯聊天就完全不碰它，也就不会被要求授权。这直接回答了"任何页面都能对话"。

### 3.3 选中文字取入 + 推荐按钮

**取选区（零授权弹窗）**，两条路，建议都做，主次分明：

1. **主：面板可见时用 activeTab 读当前选区**
   - 侧边栏获得可见/焦点（`document.visibilitychange` / `focus`）时，向后台发 `GET_SELECTION`；后台用 `chrome.scripting.executeScript` 在活动标签执行 `window.getSelection().toString().trim()`，回传选区文本。
   - `activeTab` 已授予，读当前标签选区不弹授权框。
   - ⚠️ 待验证点：side panel 场景下 `activeTab` 的授予与目标 tabId 的取法（`chrome.tabs.query({active,lastFocusedWindow})`），以及切标签后选区归属。计划编码时先写个最小验证。

2. **辅：右键菜单直达（最稳，纯零权限）**
   - 复用已有 `selection` 上下文，加"问 AI：总结选中 / 翻译选中"菜单项，点了用 `info.selectionText` 直接开一个带该选区的纯聊天会话并打开侧边栏。
   - 这条不依赖 activeTab 时序，最稳，作为兜底。

**UI（对话上方）**：

- 取到选区后，对话区顶部显示一枚「选中文本」chip（可展开预览、可清除），下面一排推荐按钮：**总结、翻译、解释、改写**（文案走 i18n，中英双语已接入）。
- 点按钮 = 发送「选中文本 + 该指令」，走 3.1 的纯聊天通路（选区当作用户消息内容，不需要抓整页 → 零授权）。
- 没有选区时，这排按钮可以退化成通用快捷 prompt（现有 `quickPrompts` 已有类似 chips，可合并复用，避免重复造）。

### 3.4 授权只在真正需要时出现

改完后授权弹窗只会在一种情况出现：用户**主动点了「总结当前页」**且该域名尚未授权。纯聊天、聊选中文字，全程零授权。

## 五、验收标准

1. 在任意普通网页（未授权域名）打开侧边栏 → AI 对话 → 直接打字发送，**全程无授权弹窗**，能正常对话。
2. 只有点「总结当前页」且域名未授权时，才弹一次授权。
3. 选中一段文字 → 对话顶部出现「选中文本」chip + 推荐按钮；点「翻译」等能把选中内容 + 指令发出去，**不弹授权**。
4. 右键选中文字 →「问 AI：总结选中」→ 打开侧边栏并带入该选区开聊。

## 六、风险与注意

- **activeTab 读选区的时序**：side panel 与 activeTab 的授予边界是这次最大不确定点，第 3 步先做最小验证，不行就以右键菜单兜底为主。
- **纯聊天 vs 总结的系统提示**：两条路系统提示不同，别让"通用聊天"误用了"总结指令"（`wrapSourceContent` 只对 `kind:'source'` 生效，需保持）。
- **不新增权限**：整套方案不申请新的 host 权限，只用已有的 `activeTab` + context menu selection。
- **遵守 CLAUDE.md**：真机验证通过并由用户明确要求后再提交（见记忆 no-auto-git-commit）。
