# 网页转 Markdown + 超级复制（SmartCopy 能力集成）

> 日期：2026-07-02
> 状态：P0 已实施（实施记录见技术方案文末），待真机验证；P1/P2 未动
> 参照产品：SmartCopy 一键全能复制 v2.3.1（Chrome 商店，151KiB）
> 配套技术方案：[2026-07-02-网页转Markdown技术方案.md](./2026-07-02-网页转Markdown技术方案.md)
> 分工：本文档定范围、分期与验收；技术细节（注入/权限/协议/管线）以技术方案为准，两边冲突时以技术方案为准

## 一、目标（第一性原理）

larksnap 目前只会导出飞书文档。用户的核心诉求是：**任何网页都能一键变成 Markdown**，顺带把 SmartCopy 的实用功能收进来，一个扩展解决"看到内容 → 拿走内容"的全部场景。

拆到最底层，其实就四件事：

1. **拿得到**：有些网站禁止选择/复制/右键，要先解锁（超级复制）
2. **转得对**：网页 HTML → 干净的 Markdown（正文提取 + 格式转换）
3. **存得下**：复制到剪贴板 / 下载成 .md 文件 / 记进历史记录
4. **顺手**：选中即复制、标签页链接复制、快捷键、浮动按钮

## 二、功能清单与分期（YAGNI：先做核心）

### P0 — 本期必做

| 功能 | 说明 |
|---|---|
| 整页转 Markdown | 当前页面正文提取 → Markdown，可复制到剪贴板或下载 .md |
| 选中内容转 Markdown | 选中一段网页内容，转成保留格式的 Markdown 复制 |
| 超级复制（解锁） | 一键解除禁止复制/禁止选择/禁止右键/禁止粘贴 |
| 选中文字自动复制 | 开关 + 最小字符数（默认 5）+ 格式（纯文本/Markdown）。**P0 只在"本标签页已手动激活 webcopy 后"生效（会话级，刷新/关页失效）**；"打开页面就自动复制"依赖 P1 的域名自动启用 |
| 标签页复制 | 复制当前标签页 / 所有标签页，首批支持：Markdown 链接、标题-URL、仅标题、仅 URL |

### P1 — 第二批

| 功能 | 说明 |
|---|---|
| 剪贴板历史 | 自动记录复制内容，上限可配（默认 10 条），历史页面查看/重复复制/删除 |
| 域名自动启用列表 | 列表内的域名打开页面就自动解锁 + 挂载功能（对应 SuperCopy 的域名列表） |
| 浮动按钮 | 页面上可拖动的超级复制按钮，可开关 |
| 快捷键 | manifest `commands`：解锁、复制当前页为 MD、开关自动复制、打开历史 |

### P2 — 暂不做（记录一下，防止过度设计）

- CSV / JSON / HTML 表格等标签页复制格式（用到再加）
- 界面多语言（larksnap 目前全中文，等国际化一起做）

## 三、技术方案（KISS：复用现有骨架）

### 3.1 HTML → Markdown 转换

- 新增依赖：`turndown` + `turndown-plugin-gfm`（HTML→MD，表格/删除线支持）、`@mozilla/readability`（整页正文提取，去导航/广告）
- 现有 `src/background/convert/markdown.ts` 是飞书 block→MD，与本功能无关，不动它
- 转换在 **content script 里跑**（DOM 就在那，Readability 需要 document）；content 只负责产出 Markdown 字符串，**写剪贴板按触发来源分流**：手势在页面里（右键菜单/快捷键/自动复制）由 content 直接写，手势在侧边栏里则把字符串回传给侧边栏写（页面无焦点时 content 写会抛 "Document is not focused"，详见技术方案 §3.5）
- 整页流程：`Readability 提取正文 → Turndown 转 MD → 头部拼上 "# 标题 + 来源 URL"`
- 选中流程：`selection.getRangeAt().cloneContents() → 序列化 HTML → Turndown`

### 3.2 超级复制（解锁）

新建 `src/content/webcopy/unlock.ts`，做三件事：

1. 捕获阶段 `stopPropagation` 拦掉页面对 `copy/cut/paste/contextmenu/selectstart/mousedown` 的封锁监听
2. 注入 CSS：`* { user-select: text !important; }`
3. 清掉 `document/body` 上的 `oncopy/onselectstart/oncontextmenu` 等内联处理器

可开可关（关闭时移除监听和样式），这是各类复制扩展的标准做法。

### 3.3 注入方式与权限（复用现有机制）

- 新 content 模块打成独立入口 `src/content/webcopy/index.ts`（vite 多入口，不混进飞书 content）
- **P0 主触发路径：右键菜单 + 快捷键命令**——这两种手势稳定授予 `activeTab`，可对任意网页 `executeScript`，无需预授权。注意：在侧边栏里点按钮**不授予** activeTab（activeTab 只在点工具栏图标/右键菜单/快捷键/omnibox 时授予），这是本方案最容易踩的坑
- **侧边栏按钮为辅助入口**：优先尝试 `executeScript`（点图标开侧边栏时授予的 activeTab 可能仍覆盖当前标签页）；失败则在按钮手势内走 `chrome.permissions.request({origins: [当前域名]})` 兜底。骨架阶段先做 15 分钟最小实验验证边界（见技术方案 §11）
- **域名自动启用**（P1）：复用现有 `chrome.permissions` 运行时授权 + `chrome.scripting.registerContentScripts` 动态注册；配置存 `webcopy.unlockDomains`，与 `trustedDomains` 平行，不混用（一个管飞书导出，一个管网页复制）

### 3.4 消息与存储

- `MSG` 新增：`WEBCOPY_PAGE_MD`、`WEBCOPY_TOGGLE_UNLOCK`、`COPY_TABS`、`CLIP_HISTORY_*`（列表/删除/清空）
- `ExtensionConfig` 新增字段：

```ts
webcopy: {
  autoCopyEnabled: boolean;      // 选中自动复制
  autoCopyMinChars: number;      // 默认 5
  autoCopyFormat: 'text' | 'markdown';
  tabCopyFormat: 'markdown' | 'title-url' | 'title' | 'url';
  unlockDomains: string[];       // 自动启用域名列表（P1）
  floatButton: boolean;          // 浮动按钮（P1）
  historyLimit: number;          // 默认 10（P1）
}
```

- 剪贴板历史新增存储键 `feishu2md:clip-history`（P1）

### 3.5 UI 入口

现状：点扩展图标直接开侧边栏（无 popup）。**推荐方案：不新开 popup**，保持交互不变：

- **侧边栏** `SidePanel.tsx` 新增「网页复制」区块：整页转 MD（复制/下载）、解锁开关、标签页复制、自动复制开关。在非飞书页面时此区块置顶（飞书导出区块自动隐藏/收起）
- **右键菜单** `contextMenus`：「复制为 Markdown」「选中内容转 Markdown」「解除复制限制」——比 popup 更顺手
- **设置页** `Options.tsx` 新增「网页复制设置」：最小字符数、格式、域名列表、历史上限、浮动按钮

manifest 变更：`permissions` 加 `contextMenus`；加 `commands` 快捷键（P1）。`clipboardWrite` 不作为必需项（content 里 `navigator.clipboard` 靠 transient activation 即可，该权限主要服务扩展页 `execCommand`，可留可不留）。

## 五、风险与对策

| 风险 | 对策 |
|---|---|
| Readability 对特殊页面（SPA、微信公众号）提取不全 | 提取失败时降级为全 body 转换；选中复制不依赖 Readability |
| 侧边栏触发时 activeTab 拿不到 / 页面无焦点写剪贴板失败 | 主触发路径放右键菜单+快捷键；侧边栏触发时由侧边栏自己写剪贴板，content 只回传字符串；骨架阶段先做最小实验验证 |
| 解锁脚本与飞书页面自身冲突 | webcopy 独立入口，飞书域名默认不自动挂载解锁 |
| 剪贴板历史属敏感数据 | 只存本地 `storage.local`，上限截断，提供一键清空；商店隐私声明同步更新 |
| 扩展体积增大 | turndown+readability 约 100KB，可接受；vite 按入口分包不影响飞书导出路径 |

## 六、验收标准

1. 在 CSDN / 知乎 / 公众号等禁复制页面：一键解锁后可正常选择复制
2. 任意文章页「整页转 Markdown」：标题、正文、图片、代码块、表格、链接正确转换，头部含来源 URL
3. 在已手动激活 webcopy 的标签页上，选中一段含粗体/链接/列表的内容 → 自动复制出合法 Markdown（未激活过的页面不生效，属预期）
4. 复制当前/全部标签页为 Markdown 链接格式，粘贴即用
5. 关闭所有开关后，扩展对普通网页零侵入（不注入任何脚本）
6. `npm run typecheck` 与 `npm run build` 通过，飞书导出功能回归正常
